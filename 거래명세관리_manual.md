# 거래명세 관리 — 사용자 매뉴얼

The Symbol 워크스페이스에서 **거래·명세** 구역의 **거래명세 관리** 화면 사용법을 정리합니다. (앱 버전에 따라 세부 문구·버튼 위치는 약간 달라질 수 있습니다.)

---

## 1. 화면 들어가기

1. 왼쪽 사이드 레일에서 **거래·명세** 구역을 선택합니다.  
2. 하위 메뉴에서 **거래명세 관리**를 누릅니다.

상단 `header` 영역에는 현재 화면 안내, 워크스페이스 요약, 그리고 **거래명세서 입력 및 월별 납품현황 관리** 안내 배너가 표시될 수 있습니다. 본문 작업은 그 아래 패널에서 진행합니다.

---

## 2. 저장 방식 (로컬 / 클라우드)

- **로컬(개인 브라우저)**: 명세 데이터가 이 브라우저에 저장되는 방식입니다. 다른 PC·브라우저와 자동으로 맞춰지지 않을 수 있습니다.  
- **클라우드(회사 문서)**: 로그인한 회사(워크스페이스) 기준으로 문서가 동기화되는 방식입니다. 팀과 같은 데이터를 이어서 쓸 수 있습니다.

화면에 표시되는 **저장 상태**(저장 중 / 저장됨 등)는 거래명세 입력 모달 상단에서 확인할 수 있습니다.

---

## 3. 거래명세서 입력 (모달)

납품 건을 새로 넣거나 수정할 때는 **거래명세서 입력** 버튼을 눌러 모달을 엽니다.  
(입력 목록에서 행을 수정하거나, 최근 납품을 복제할 때는 모달이 자동으로 열릴 수 있습니다.)

### 3.1 입력 항목 요약

| 항목 | 설명 |
|------|------|
| **납품일** | 실제 납품 기준일 |
| **횟수** | 같은 날·같은 거래처 기준 납품 차수 등 |
| **거래처명** | 목록에서 선택하거나 직접 입력 |
| **품목** | 목록 선택 또는 직접 입력. **여러 품목**으로 한 번에 여러 줄을 넣을 수 있습니다(거래처 선택 후). |
| **규격/단위** | 빠른 선택 버튼 또는 직접 입력 |
| **수량·단가** | 숫자 입력. 거래처별 단가·품목 마스터가 있으면 단가가 자동 제안될 수 있습니다. |
| **과세구분** | 부가세 별도 / 없음 등 |

- **저장**: 현재 폼 내용을 명세 목록에 반영합니다.  
- **수정 중**인 경우에는 안내에 따라 **저장** 또는 **수정 취소**를 사용합니다.  
- **닫기**·배경(딤) 클릭·**Esc**로 모달을 닫을 수 있습니다(다른 작은 창이 위에 떠 있을 때는 그 창을 먼저 닫아야 할 수 있습니다).

### 3.2 관리자 모드 (단가·품목 마스터)

- 모달 상단의 **관리자 모드**는 **4자리 PIN** 확인 후 열립니다. (입출고 화면의 전체 초기화 등과 동일한 PIN 정책을 따릅니다.)
- 열리면 **거래처별 단가** 편집, **품목 마스터(공통 기본 단가)** 관리를 할 수 있습니다.
- 일반 입력만 할 때는 관리자 모드를 닫아 두어도 됩니다.

---

## 4. 본문 탭: 거래처 카드 / 입력 목록 / 캘린더 / 월별 납품현황

패널 상단에서 보기 방식을 바꿉니다.

### 4.1 거래처 카드

거래처별로 요약된 카드 형태로 볼 수 있습니다. (카드에서 입력 목록으로 연결되는 동작이 있을 수 있습니다.)

### 4.2 입력 목록

- **거래명세서 입력** 버튼: 위에서 설명한 입력 모달을 엽니다.  
- **연도 선택**: 집계·표시에 쓰이는 연도를 바꿉니다.  
- **입력목록 엑셀 저장** / **월별현황 엑셀 저장**: 각각 목록·월별 요약을 스타일이 적용된 엑셀로 내려받을 수 있습니다. (브라우저에서 **저장 폴더**를 지정해 두면 저장 위치를 맞추기 쉽습니다.)

**입력 목록** 탭을 처음 열면 기간 필터가 **이번달**로 잡히도록 되어 있는 경우가 많습니다.

- **검색**: 거래처·품목·메모 등으로 좁힙니다.  
- **기간 칩**: 최근 7일 / 이번달 / 선택 연도 / 전체.  
- **과세 칩**: 전체 / 부가세 별도 / 없음 등으로 걸러 봅니다.  
- 목록에서 **수정·삭제·복제** 등 액션을 사용할 수 있습니다(화면에 표시된 버튼 기준).

### 4.3 캘린더

납품일 기준으로 일정을 달력 형태로 확인합니다.

### 4.4 월별 납품현황

연도·거래처 기준 월별 집계를 표로 봅니다. 엑셀 저장과 함께 쓰면 보고용으로 편합니다.

---

## 5. 거래명세표 (미리보기·양식)

하단 **거래명세표** 영역에서는 실제 인쇄에 가까운 형태로 묶인 문서를 미리 보거나, **엑셀 양식**에 맞춰보낼 수 있습니다.

- **[거래명세표 미리보기]**: 접었다 펼칠 수 있습니다. 기본은 접혀 있을 수 있습니다.  
- **문서 선택**: 납품일·거래처·회차 등으로 묶인 시트를 고릅니다.  
- **양식 업로드 / 수정 / 저장**: 회사에서 쓰는 거래명세표 엑셀 양식을 연결한 뒤, 선택한 건을 양식에 채워 저장할 수 있습니다.  
- **거래명세표 양식 정보**: 사업자번호·상호·주소·계좌 등 자주 바뀌는 칸만 화면에서 고칠 수 있는 영역입니다.

양식이 없거나 묶을 데이터가 없으면 안내 문구가 표시됩니다.

---

## 6. 자주 묻는 것

**Q. 입력했는데 월별 현황에 안 보여요.**  
A. 납품일·연도 필터·거래처 조건을 확인해 보세요. 저장이 완료되었는지(저장 상태)도 함께 확인합니다.

**Q. 다른 PC와 숫자가 달라요.**  
A. 로컬 모드면 브라우저마다 데이터가 분리됩니다. 클라우드 모드·같은 회사 로그인인지 확인합니다.

**Q. 엑셀 양식 저장이 실패해요.**  
A. 브라우저 권한(파일 저장·폴더 지정), 양식 파일 형식(.xlsx), 그리고 선택한 납품·거래처에 줄 데이터가 있는지 확인합니다.

**Q. 미리보기·앱의 「계」와 엑셀 파일의 공급가액·세액 합이 1원씩 다를 때가 있어요.**  
A. 앱은 **품목마다** 공급가의 10%를 내림(`floor`)하고, 합계 금액 끝자리가 1이 되면 세액에서 1원을 빼는 보정을 합니다. 반면 엑셀 양식에는 예전부터 `SUM(공급)` + `ROUND(SUM(공급)×10%)`처럼 **한 번에** 세액을 구하는 수식이 들어 있는 경우가 많아, 줄 합과 1원 어긋날 수 있습니다. 저장 시 **25행**의 공급·세액 합계 칸(M·P 및 복제 블록 AF·AI)에 앱과 같은 숫자를 직접 넣도록 맞춰 두었습니다. **커스텀 양식**에서 합계 행이 25행이 아니면 코드의 `STATEMENT_TEMPLATE_FOOTER_ROW`를 양식에 맞게 바꿔야 합니다.
