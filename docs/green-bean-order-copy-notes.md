# 생두 주문 페이지 카피·구조 정리 메모

UI 문구 중복·과다 설명 점검 결과(2026). 1번부터 순서대로 손볼 수 있음.

## 1. 같은 「합계」가 여러 층에 반복

- **히어로**: 현재 주문표 기준 수량·총액(실시간).
- **월별 추이 카드**: 가장 최근 **월 기록** 스냅샷 기준 수량·총액.
- **저장된 월별 기록 표**: 달마다 기록된 수량·총액.

라벨이 비슷하면 “같은 숫자 두 번”으로 느껴질 수 있음 → **출처를 라벨에 분명히** (주문표 vs 월 기록).

## 2. 월별 섹션 안 설명 문장 겹침

- 패널 헤더 `muted`와 `기록할 달` 옆 힌트가 모두 “월에 기록 / 달 맞춤” 주제.

→ **구조**: 헤더 설명 제거(제목만). 힌트는 **입력에 안 보이는 규칙만**(엑셀→달 자동, 같은 달 덮어쓰기). “원두별 그래프에 쓰인다”는 **원두별 섹션**에서만 설명(힌트와 중복 제거).

## 3. 기록 없을 때 안내 이중

- 차트 빈 상태 문구와 `저장된 월별 기록` 빈 상태 문구가 유사.

→ **2번과 함께 처리**: `chartRows`와 표 행이 같은 데이터인데, 기록 없을 때 **같은 행동 안내를 두 블록에 반복하지 않음**. 기록이 없으면 안내는 **한 블록**만, `저장된 월별 기록` 표는 **첫 기록 이후에만** 노출.

## 4. 원두별: 스냅샷 없을 때 긴 설명 + 다른 곳과 겹침

- 품목 스냅샷이 없을 때 안내 문단이 길고, 월 힌트와 행동 안내가 겹침.

→ 짧게 한 줄 + 필요 시 접기.

## 5. 원두별: 리스트와 칩 이중 표시

- 왼쪽 랭킹과 오른쪽 포커스 칩에 같은 품목이 두 번.

→ 칩 제거 또는 리스트만 강조 등 선택.

## 6. 히어로 vs 주문표 패널: 엑셀·기준·비교 이야기 겹침

- 상단 카피와 주문표 패널 설명이 주제가 비슷.

→ 한 블록에만 긴 설명.

## 7. 일일 원두 가격: 버튼 줄에 설명이 길게 나열

- 시각·차이 열·DEV 안내가 한 줄에 몰림.

→ 접기/한 줄 요약.
